## Residual risks / unknowns

- Truly free-form bare names such as `John Smith needs help with eviction`
  remain intentionally unsupported because deterministic matching would create
  high false-positive risk on ordinary legal text.
- Name detection still assumes name-like capitalization; lowercase, heavily
  obfuscated, or otherwise noisy free-form names may evade redaction.
- Multilingual coverage is expanded for Spanish contextual phrases only; other
  languages were not added in this remediation.
- Idaho DL detection is intentionally gated to explicit license context and the
  audited Idaho shape `[A-Z]{2}\d{6}[A-Z]`; generic alphanumeric identifiers
  without license context continue to pass through.

## Rollback notes

- Code rollback: revert the `PiiRedactor` and PHPUnit/kernel/doc changes listed
  above.
- No schema migration, config export, or data backfill was introduced, so no
  database rollback is required.
- If the new contextual patterns prove too aggressive in production-like text,
  rollback is isolated to the redactor/test/doc bundle for `RAUD-10`.

## Still-unverified surfaces

- Additional multilingual phrasing outside the new English/Spanish fixtures.
- Non-capitalized or adversarially obfuscated free-form personal names.
- Non-Idaho or differently formatted license/ID numbers that were not part of
  the audited scope.
