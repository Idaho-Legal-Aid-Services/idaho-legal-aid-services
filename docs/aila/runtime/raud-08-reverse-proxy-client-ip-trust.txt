# RAUD-08 Reverse-Proxy / Client-IP Trust Remediation Report

- Date: 2026-03-09
- Prompt ID: `RAUD-08`
- Finding IDs: `F-05`, `N-05`
- Prior status: Open / unverified. The repo had no explicit reverse-proxy trust declaration, `/assistant/api/message` and `/assistant/api/track` keyed flood control directly from `Request::getClientIp()`, and Pantheon `dev`/`test`/`live` read-only checks reported `reverse_proxy=NULL`, `reverse_proxy_addresses=null`, `reverse_proxy_trusted_headers=NULL`, and empty Symfony trusted proxies.
- Post-change status: Partially Fixed
- Verification level tag: `Partially Fixed`

## Evidence

- Pre-edit repo declaration check:
  - Command summary: `rg -n "trusted_(host|proxy)|reverse_proxy|X-Forwarded-For|Forwarded|getClientIp"` across the cited settings/controller/docs surfaces.
  - Result: only Drupal default reverse-proxy comments were present in `default.settings.php`; assistant flood call sites in `AssistantApiController.php` used `Request::getClientIp()` directly.
- Post-edit repo declaration check:

## Residual risks / unknowns

- Pantheon has not yet deployed the RAUD-08 code, so read-only environment checks still reflect the old runtime.
- Even after deploy, the trust chain remains incomplete until `ILAS_TRUSTED_PROXY_ADDRESSES` is set to the actual upstream proxy IPs/CIDRs for each environment.
- No authenticated HTTP capture of `/assistant/api/health` or `/assistant/api/metrics` was collected from Pantheon in this session, so live request-context `proxy_trust` output remains unverified.
- If operators misconfigure the allowlist, flood identity may collapse to `REMOTE_ADDR` or a proxy mismatch warning state; the code stays fail-closed but operational validation is still required.

## Rollback notes

- Code rollback: revert the RAUD-08 code/doc changes listed above.
- Runtime rollback after deploy: unset `ILAS_TRUSTED_PROXY_ADDRESSES` and rebuild caches; the code reverts to fail-closed `REMOTE_ADDR` behavior when the variable is absent.
- No `settings.pantheon.php` or `pantheon.yml` rollback is needed because those files were intentionally left unchanged.
