2. Exact enablement steps
-------------------------

Repo/runtime controls

- Added runtime-only `ILAS_VECTOR_SEARCH_ENABLED` handling in
  `web/sites/default/settings.php`.
- Kept sync config safe-by-default with `vector_search.enabled=false`.
- Forced `live` back to `FALSE` even if the runtime toggle is set.
- Extended `AssistantSettingsForm` so `live` displays vector enablement as
  forced off and coerces submissions back to `FALSE`.
- Updated runtime-truth reporting so `vector_search.enabled` reports the real
  authority channel (`config export`, runtime toggle, or `live` branch).
- Added promptfoo provenance metrics and the repo-owned
  `scripts/ci/run-vector-provenance-smoke.js` helper.
- Added language-path filtering in `FaqIndex` and `ResourceFinder` so enabled
  vector retrieval does not surface cross-language translation ghosts.
- Fixed the `dev` / `test` precedence bug in `settings.php`: the private flag
  file is now checked whenever vector search is still effectively disabled,
  not only when the secret is absent.

Local repair and enablement

1. Ran `ddev drush updatedb -y` to apply `10008`, `10009`, and `10010`.
2. Imported the local Pinecone/retrieval parity config with partial
   `config:import`.
3. Rebuilt caches with `ddev drush cr`.
4. Confirmed:
   - `retrieval.*` was populated again
   - lexical indexes were present again
   - `key.key.gemini_api_key` used `ilas_runtime_site_setting`
   - `ai_vdb_provider_pinecone.settings` contained
     `query_connect_timeout_seconds=1.0` and
     `query_request_timeout_seconds=2.0`
5. Enabled local vector supplementation only through `.ddev/.env` by setting
   `ILAS_VECTOR_SEARCH_ENABLED=1`, then restarted DDEV.

Hosted rollout

- Published the `test` precedence fix as GitHub PR `#22`
  (`Allow test vector flag file override`).
- PR `#22` passed `Analyze (actions)`, `Analyze (javascript-typescript)`,
  `CodeQL`, `GitGuardian Security Checks`, `PHPUnit Quality Gate`, and
  `Promptfoo Gate`.
- PR `#22` merged to `github/master` at `2026-03-17T21:54:18Z` with merge
  commit `1d0561dd1d7a48c7bf3b5677594c8ebed903b854`.
- Pushed the same one-line precedence fix to Pantheon `origin/master` before
  merge so the non-live verification could proceed without waiting on the
  hosted gate.
- Verified Pantheon `dev` first on release `5e6e879`, then resynced Pantheon
  `origin/master` to merged release `1d0561d`.
- Promoted code from `dev` to `test`; verified Pantheon `test` on release
  `test_160`.
