1. Prior status
---------------

Before TOVR-09, the repo already showed the Pinecone/vector stack in config:

- `config/search_api.server.pinecone_vector.yml` defines Search API AI server
  `pinecone_vector` with:
  - database `ilas-assistant`
  - collection `default`
  - metric `cosine_similarity`
  - embeddings engine `gemini__models/gemini-embedding-001`
  - dimensions `3072`
- `config/search_api.index.faq_accordion_vector.yml` and
  `config/search_api.index.assistant_resources_vector.yml` define the FAQ and
  resource vector indexes on that server.
- `config/key.key.pinecone_api_key.yml` intentionally stores an empty config
  value, and `web/sites/default/settings.php` injects `ILAS_PINECONE_API_KEY`
  into effective runtime config.
- `config/ilas_site_assistant.settings.yml` keeps:
  - `retrieval.faq_vector_index_id=faq_accordion_vector`
  - `retrieval.resource_vector_index_id=assistant_resources_vector`
  - `vector_search.enabled=false`
  - `vector_search.fallback_threshold=2`
  - `vector_search.min_vector_score=0.7`
  - `vector_search.score_normalization_factor=100`
  - `vector_search.min_lexical_score=0`

Known pre-TOVR-09 gaps:

- TOVR-08 originally recorded Pantheon `ilas:runtime-truth` as deployment-gated.
- The production audit still described Pinecone as disabled and partially
  unverified.
- Existing docs proved config presence and secret wiring, but did not yet give
  one current per-environment answer for:
  - secret presence
  - index enablement
  - population
  - queryability
  - active runtime callers and blockers

Five mandatory environment questions

1. Which environments actually have Pinecone secrets?
   - Answer: all four sampled environments (`local`, `dev`, `test`, `live`)
     currently report Pinecone key presence in effective runtime.

2. Which environments have enabled vector indexes?
   - Answer: all four sampled environments have enabled vector server/index
     entities, but `local` is drifted because the lexical indexes expected by
     live retrieval code are missing there.

3. Are the vector indexes populated and searchable?
   - Answer:
     - `dev`, `test`, and `live`: yes, populated and queryable by read-only
       sample searches.
     - `local`: populated but query-time execution is blocked by local active
       config drift and pending updates.

4. Is vector search runtime-enabled anywhere?
   - Answer: no. `vector_search.enabled=false` in effective runtime on
     `local`, `dev`, `test`, and `live`.

5. Which code paths and feature flags gate current use?
   - Answer:
     - `FaqIndex::search()` and `ResourceFinder::findByTypeSearchApi()` run the
       lexical Search API branch first.
     - `FaqIndex::supplementWithVectorResults()` and
       `ResourceFinder::supplementWithVectorResults()` only invoke vector search
       when `vector_search.enabled=true` and lexical results are sparse or below
       the configured score threshold.
     - Current live callers are in `AssistantApiController`:
       - early retrieval before fallback gate evaluation
       - public suggest endpoint FAQ suggestions
       - public FAQ endpoint
       - FAQ intent branch
       - forms, guides, and generic resources branches
       - same-area follow-up resource branch
       - final default FAQ/resource fallback branch
     - Because `vector_search.enabled=false` everywhere, none of those current
       code paths will actually call the vector indexes in normal runtime today.

Stack reconstruction

- Pinecone Search API server:
  - `pinecone_vector` uses Search API AI Search with Pinecone as the database,
    Gemini models for chat + embeddings, cosine similarity, and 3072
    dimensions.
- Vector indexes:
  - `faq_accordion_vector` indexes `faq_item` and `accordion_item` paragraphs.
  - `assistant_resources_vector` indexes `resource` nodes.
- Secret wiring:
  - Pinecone API key: `ILAS_PINECONE_API_KEY` -> `settings.php` runtime config
    injection into `key.key.pinecone_api_key`.
  - Gemini API key for query-time embeddings:
    `ILAS_GEMINI_API_KEY` -> `settings.php` runtime site setting
    `ilas_gemini_api_key` -> `key.key.gemini_api_key`.
  - Vertex AI service-account fallback:
    `ILAS_VERTEX_SA_JSON` -> runtime site setting -> `key.key.vertex_sa_credentials`.
- Assistant retrieval config:
  - Hosted `dev` / `test` / `live` all reported:
    - `retrieval.faq_index_id=faq_accordion`
    - `retrieval.resource_index_id=assistant_resources`
    - `retrieval.resource_fallback_index_id=content`
    - `retrieval.faq_vector_index_id=faq_accordion_vector`
    - `retrieval.resource_vector_index_id=assistant_resources_vector`
  - Hosted `dev` / `test` / `live` all reported:
    - `vector_search.enabled=false`
    - `vector_search.fallback_threshold=2`
    - `vector_search.min_vector_score=0.7`
    - `vector_search.score_normalization_factor=100`
    - `vector_search.min_lexical_score=0`
